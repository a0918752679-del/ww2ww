# V4 功能說明

## 經營模式

本系統採「會員商城 + 線上一番賞」模式，不建議設計成單純付費抽獎。較安全的營運設計是：客戶購買商品或組合包後，獲得刮刮卡資格。

## 庫存邏輯

1. 客戶建立訂單時只檢查商品庫存，不立即扣減。
2. 付款確認後，商品庫存扣減，刮刮卡發放。
3. 客戶刮卡中獎時，獎項庫存扣減。
4. 獎項剩餘數量為 0 時，自動標示 Sold out，不再抽出該獎項。
5. 中獎後需填寫寄件資料，進入後台出貨管理。

## 建議正式營運加強

- 串接正式金流簽章驗證。
- Google / Apple OAuth callback 完整化。
- PostgreSQL 取代 Google Sheet 作為主資料庫。
- 後台人員權限分級。
- 活動辦法、個資告知與付款條款頁面。
- 抽獎機率與庫存揭露頁面。
